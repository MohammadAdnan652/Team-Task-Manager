# Deployment Guide - Team Task Manager

## 🚀 Railway Deployment

### Prerequisites

1. GitHub Account with code pushed
2. Railway Account (https://railway.app)
3. MongoDB Atlas Account (free tier)

---

## Step 1: MongoDB Atlas Setup (Database)

1. Go to https://www.mongodb.com/cloud/atlas
2. Create free account
3. Create a new project and cluster
4. Click "Connect" → "Drivers"
5. Copy connection string (replace `<password>` and `<username>`)
6. Use this as `MONGODB_URI` in Railway

**Example connection string:**
```
mongodb+srv://username:password@cluster0.xgz1q.mongodb.net/team-task-manager?retryWrites=true&w=majority
```

---

## Step 2: Backend Deployment on Railway

1. Go to https://railway.app
2. Click "New Project"
3. Select "Deploy from GitHub"
4. Choose your repository
5. Select the `/backend` directory as the root
6. Railway will auto-detect Node.js

### Configure Environment Variables

In Railway Dashboard → Variables:

```
PORT=8000
NODE_ENV=production
MONGODB_URI=mongodb+srv://username:password@cluster0...
JWT_SECRET=generate_a_random_string_here
FRONTEND_URL=https://your-frontend-url.railway.app
```

#### Generate JWT_SECRET

Run this in your terminal:
```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

### Build & Start Commands

Railway will auto-detect from `package.json`:
- Build: `npm install`
- Start: `npm start` (from backend/package.json)

**Automatic Deployment:** Push code to `main` branch and Railway automatically redeploys

**Get Backend URL:** Once deployed, Railway provides a public URL like:
```
https://team-task-manager-backend.railway.app
```

---

## Step 3: Frontend Deployment on Railway

1. Create another new Railway project
2. Select "Deploy from GitHub"
3. Choose same repository
4. Select `/frontend` as root directory

### Configure Environment Variables

```
VITE_API_URL=https://your-backend-railway-url/api
```

Replace with actual backend URL from Step 2.

### Build & Start Commands

Railway auto-detects:
- Build: `npm run build`
- Start: `npm run preview`

---

## Step 4: Update Backend FRONTEND_URL

Update Railway backend environment:
- `FRONTEND_URL=https://your-frontend-url.railway.app`

This must match your frontend Railway URL exactly.

## Troubleshooting

### CORS Errors
- Verify `FRONTEND_URL` in backend matches frontend URL exactly
- Check Network tab in browser DevTools

### 502 Bad Gateway
- Check Railway logs: "View Logs" in project
- Verify database connection string is correct
- Check all environment variables are set

### Token Issues
- Clear browser cookies/localStorage
- Login again
- Check JWT_SECRET is set in backend

### Database Connection Error
- Verify MongoDB Atlas IP whitelist includes Railway
- Check connection string for typos
- Verify username/password in connection string

### Build Fails
- Check Railway logs for errors
- Ensure package-lock.json is in repo
- Verify Node version compatibility

## Security Checklist

- [ ] JWT_SECRET is strong and random
- [ ] MongoDB password is complex
- [ ] CORS only allows frontend domain
- [ ] Environment variables not committed to git
- [ ] HTTPS enabled (Railway provides by default)
- [ ] IP whitelist on MongoDB Atlas
- [ ] No API keys in code
